# How to Troubleshoot No Network Access

## Purpose
This guide explains basic checks when a user reports no network access.

## Steps
1. Confirm whether the issue affects one device or multiple devices.
2. Check whether the network adapter is enabled.
3. Verify IP configuration using command-line tools such as `ipconfig`.
4. Test connectivity to the local gateway.
5. Test name resolution if DNS is suspected.
6. Restart the adapter or device if needed.
7. Escalate the issue if the problem is outside first-level support scope.

## Result
These steps help identify whether the problem is local, network-related, or requires escalation.

## Notes
Always document the checks performed and the final result in the ticket.
