## Purpose

This mini-report documents a simulated first-level IT support incident involving a suspected phishing email.

The goal is to show how IT support can document the report, collect evidence, guide the user safely, escalate when needed, and record the outcome.

---

## Incident Summary

| Field | Details |
|---|---|
| Incident Type | Suspected phishing email |
| Affected Service | Email / Microsoft 365 |
| Affected User | End user |
| Priority | Medium |
| Status | Resolved / Escalated |
| Escalation Required | Yes, if link was clicked or credentials were entered |
| Support Level | First-level IT support |

---

## User Reported Issue

The user reported receiving a suspicious email asking them to click a link and confirm account information.

The user was unsure whether the email was legitimate and contacted IT support before taking further action.

---

## Symptoms Observed

- Sender address looked unusual or unexpected.
- Email created urgency or pressure.
- Message asked the user to click a link.
- Email requested account verification or password-related action.
- Link destination did not clearly match the expected company or service.
- User was unsure whether the message was safe.

---

## Initial Questions Asked

- Did you click the link?
- Did you open any attachment?
- Did you enter your password or MFA code?
- Did you reply to the email?
- When did you receive the message?
- Is the email still in your mailbox?
- Did anyone else receive the same message?

---

## Actions Taken

1. Instructed the user not to click links or open attachments.
2. Asked the user not to delete the email until evidence was captured.
3. Collected sender, subject, timestamp, and screenshot.
4. Checked whether the user clicked the link or entered credentials.
5. Advised the user to report the email according to company process.
6. Escalated to security/admin team if user interacted with the message.

---

## Root Cause

The likely cause was a phishing attempt designed to trick the user into opening a link or sharing account information.

---

## Resolution

The incident was handled by:

- Confirming the user did not interact with the message.
- Collecting evidence.
- Advising the user not to click links or attachments.
- Reporting or escalating the email for review.
- Documenting the incident in the ticket.

---

## Verification

The case is complete when:

- User confirmed whether they clicked the link or not.
- Evidence was captured.
- Suspicious email was reported or escalated.
- User received safe guidance.
- Ticket notes were completed.

---

## Escalation Criteria

Escalate immediately if:

- User clicked the link.
- User entered password or MFA code.
- User opened a suspicious attachment.
- Multiple users received the same email.
- The email appears to impersonate company systems.
- Malware or account compromise is suspected.

---

## Evidence to Capture

Recommended evidence:

- Screenshot of the email
- Sender address
- Subject line
- Date and time received
- Link preview / visible URL, if safe to capture
- User statement: clicked / not clicked
- Attachment name, if present
- Ticket number

---

## Ticket Note Example

**Issue:** User reported suspected phishing email.  
**Action:** Advised user not to click links or open attachments, collected sender/subject/timestamp, confirmed whether user interacted with the email, and escalated if needed.  
**Result:** Email reported for review. User confirmed no credentials were entered / escalation required due to user interaction.  
**Status:** Resolved / Escalated.  
**Escalation:** Required if link was clicked, credentials were entered, or multiple users are affected.

---

## User Communication Example

Hello, thank you for reporting the suspicious email. Please do not click any links, open attachments, or reply to the message. We have documented the details and will forward/escalate it for review. If you already clicked the link or entered your password, please tell IT support immediately.

---

## Prevention Notes

- Do not click links in unexpected emails.
- Check sender addresses carefully.
- Be careful with urgent password or account verification messages.
- Report suspicious emails instead of deleting them immediately.
- Contact IT support if you clicked a suspicious link or entered credentials.

## Summary

This incident report shows how first-level IT support can handle a suspected phishing email.
