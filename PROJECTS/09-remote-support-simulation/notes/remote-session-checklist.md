# Remote Session Checklist

## Purpose

This checklist documents the basic steps that should be followed before, during, and after a remote support session.

It is intended for first-level IT support situations where a supporter connects to a user device using an approved remote support tool.

---

## 1. Before Starting the Session

- [ ] Confirm the user request.
- [ ] Clarify the affected device.
- [ ] Clarify what the user is trying to do.
- [ ] Decide whether remote support is appropriate.
- [ ] Use only an approved remote support tool.
- [ ] Ask for user consent before starting remote access.
- [ ] Explain that the user should stay present during the session.
- [ ] Do not ask the user to share passwords.

---

## 2. Starting the Session

- [ ] Start the remote support session from the support technician device.
- [ ] Provide the session code, link, or connection request as required by the tool.
- [ ] Wait for the user to accept the session.
- [ ] Confirm that the correct user device is visible.
- [ ] Confirm that the user can see what the supporter is doing.

---

## 3. During the Session

- [ ] Explain intended support actions before making changes.
- [ ] Only check areas relevant to the support request.
- [ ] Avoid opening private files, personal folders, or unrelated applications.
- [ ] Do not save or copy user data unless explicitly required and approved.
- [ ] Keep the user informed during the session.
- [ ] If the issue cannot be solved, explain the next step.

---

## 4. Ending the Session

- [ ] Confirm whether the user can continue working.
- [ ] Close any tools or windows opened during the support session, if appropriate.
- [ ] End the remote session safely.
- [ ] Confirm that remote access is no longer active.
- [ ] Remind the user not to reuse or share session codes or passwords.

---

## 5. After the Session

- [ ] Document what was checked.
- [ ] Document whether anything was changed.
- [ ] Document the final result.
- [ ] Document whether escalation is required.
- [ ] Remove or anonymise sensitive information before publishing screenshots or notes.
