## Purpose

This knowledge base article explains how to restore an earlier version of a file by using OneDrive version history.

The goal is to help users recover a previous file version when a file was changed, overwritten, or edited by mistake.

---

## Symptoms

A user may need this guide when:

- A file was changed by mistake.
- Important content was deleted from a document.
- A newer version of a file is incorrect.
- A file was overwritten.
- The user wants to compare earlier versions of a file.
- The user needs to restore a previous version without creating a new file manually.

---

## Before You Start

Make sure:

- You are connected to the internet.
- You are signed in to the correct Microsoft 365 account.
- The file is stored in OneDrive.
- You know the name or location of the affected file.
- You do not delete the current file before checking version history.

---

## Step 1 — Open OneDrive Online

1. Open your web browser.
2. Go to OneDrive online.
3. Sign in with your Microsoft 365 work or school account.
4. Find the affected file.

If you cannot find the file, check the correct OneDrive folder or contact IT support.

---

## Step 2 — Open Version History

1. Right-click the affected file.
2. Select **Version history**.
3. Wait for the list of previous versions to load.

OneDrive will show earlier saved versions of the file if version history is available.

---

## Step 3 — Review Previous Versions

1. Look through the available versions.
2. Check the date and time of each version.
3. If available, check who modified the file.
4. Open or preview the version before restoring it.

Choose the version that was saved before the unwanted change happened.

---

## Step 4 — Restore the Correct Version

1. Select the version you want to restore.
2. Choose **Restore**.
3. Confirm the action if prompted.
4. Wait for OneDrive to restore the selected version.

The restored version becomes the current version of the file.

---

## Step 5 — Verify the File

After restoring the file:

1. Open the file.
2. Check that the correct content is restored.
3. Confirm that the file opens normally.
4. Save the file if needed.
5. Inform IT support if the restored version is still incorrect.

---

## Important Notes

- Restoring a previous version does not usually delete the version history.
- The restored version becomes the newest version.
- Version history is only available if the file was saved in OneDrive or SharePoint.
- Very old versions may not always be available, depending on retention settings.
- If the file was deleted, check the OneDrive recycle bin instead.

---

## When to Contact IT Support

Contact IT support if:

- You cannot find the file.
- Version history is not available.
- You restored the wrong version.
- The file is missing completely.
- The file was deleted from OneDrive.
- You do not know which version to restore.
- The file contains important business data and you are unsure what to do.

When contacting IT support, include:

- The file name.
- The OneDrive folder location.
- The date and time when the file was changed.
- The version you tried to restore.
- A screenshot of any error message.

---

## User Communication Example

**User message to IT support:**

Hello, I need help restoring an earlier version of a OneDrive file. The file name is `<file name>`. It was changed by mistake on `<date/time>`. I checked version history, but I am not sure which version to restore / I tried restoring a version, but the problem is still there.

---

## Summary

OneDrive version history helps users recover earlier versions of files stored in OneDrive.

Users should open OneDrive online, find the file, open version history, review the available versions, restore the correct version, and verify that the file content is correct.

If the correct version is missing or the user is unsure what to restore, IT support should review the file history and help prevent accidental data loss.
