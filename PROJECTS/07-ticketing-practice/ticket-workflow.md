# Ticket Workflow

This project follows a simple first-level support ticket workflow:

**Problem → Analysis → Troubleshooting Steps → Resolution / Escalation → Result**

## Workflow Used

1. The user reports an issue.
2. A ticket is created.
3. Category, priority, requester, and assignee are set.
4. Troubleshooting notes are documented as internal notes.
5. User-facing updates are added as public comments when needed.
6. The ticket is either resolved, closed, or escalated/waiting.
7. The final status is documented.

## Documentation Focus

Each ticket case documents:

- the user problem
- the support analysis
- troubleshooting steps
- resolution or escalation
- final result
