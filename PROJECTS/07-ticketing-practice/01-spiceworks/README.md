# Spiceworks Ticketing Practice

---

## Overview

This project demonstrates basic helpdesk ticket handling using Spiceworks Core Plan. The goal was to practice how IT support tickets are created, categorized, documented, updated, resolved, and closed.

---

## Tools Used

- Spiceworks Help Desk
- Windows 10/11 practice environment
- Browser-based ticketing interface

---

## Practiced Ticket Scenarios

1. Browser-based application does not open
2. User cannot access the internet
3. User cannot access shared folder

---

## Skills Demonstrated

- Ticket creation
- Incident categorization
- Priority assignment
- Troubleshooting documentation
- Status updates
- Resolution notes
- Escalation awareness
- User communication

---

## Ticket Workflow

The project follows a basic helpdesk lifecycle:

1. Issue intake
2. Ticket creation
3. Categorization
4. Troubleshooting notes
5. Resolution or escalation
6. User confirmation
7. Ticket closure
